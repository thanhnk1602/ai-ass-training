# Project Context (Sample)

This is a sample CLAUDE.md showing how Memory and CLAUDE.md complement each other.

## What Goes in CLAUDE.md (Static)
- Project description and purpose
- Tech stack overview
- Code conventions and style guide
- Folder structure
- Build and deploy commands

## What Goes in Memory (Dynamic)
- Architectural decisions discovered during development
- Debugging insights ("X doesn't work because of Y")
- Team preferences learned from code review feedback
- API quirks and workarounds
- Performance findings and optimizations

Together, CLAUDE.md + Memory = Complete AI context.
